**Section 6: Display Patterns**

6.1 Visual Hierarchy:
- Icon: Optional, 48px, muted color, relevant to context
- Title: text-lg, font-semibold, text-primary
- Description: text-sm, text-secondary, max 2 lines
- CTA: Primary button for main action, link style for secondary

6.2 Layout Guidelines:
- Center-aligned for full-page empty states
- Left-aligned for inline/card empty states
- Vertical spacing: icon → 16px → title → 8px → description → 16px → CTA

6.3 Icon Recommendations:
| Context | Icon Suggestion |
|---------|-----------------|
| No items | MagnifyingGlass or Folder |
| No search results | MagnifyingGlass |
| No messages | ChatCircle |
| No notifications | Bell |
| First time | Sparkle or RocketLaunch |
| Error loading | WarningCircle |

**Section 7: Implementation Guidelines**

7.1 Component Structure:
```tsx
interface EmptyStateProps {
  icon?: React.ReactNode;
  title: string;
  description?: string;
  action?: {
    label: string;
    onClick: () => void;
  };
}
```

7.2 Accessibility:
- Use role="status" for dynamic empty states
- Include aria-label on icons
- Ensure CTA is keyboard accessible

7.3 State Detection:
- Check data.length === 0 vs loading vs error
- Distinguish first-time vs filtered empty
- Handle search vs browse context

**Section 8: Quick Reference Card**

**Always Do:**
- Include a helpful description
- Provide a CTA when action is possible
- Match module tone (Consumer warm, Business efficient, Admin factual)
- Use sentence case throughout
- End descriptions with a period

**Never Do:**
- Leave empty state without any text
- Use "Nothing here" alone (too vague)
- Make user feel bad ("You have no friends")
- Use technical language in consumer module
- Skip the CTA when one is available
